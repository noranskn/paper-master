# Paper to Master Table

## 适用场景

在以下情况使用本 skill：

- 用户阅读 **PDF 论文**，希望：
  - 提取：标题（含中文翻译）、作者机构、发表年份、期刊/会议及分区等级、摘要、研究方法与内容、实验、相关工作、自我思考、有无代码
  - 按固定字段整理为一行，**追加到总表第一张表的最后一行**
- 总表可以是 `SUMMARY.xlsx`（推荐）或任意其他 xlsx/csv 文件的 **第一张表**

---

## 总表字段定义（按顺序）

总表每一行对应一篇论文，字段推荐顺序如下（可与现有表头自动对齐）：

1. **标题**  
   - 形式：`英文标题`（实际换行）`（中文翻译）`
   - 标题中英文翻译之间必须是"真实换行"（用于表格自动换行，等同 Excel 的 `Alt+Enter`）
   - 若你发现 Excel 单元格仍显示为单行，请在该列上开启 `自动换行`（Wrap Text）

2. **作者（机构/学校）**  
   - 只写机构/学校，不写个人姓名  
   - 多个机构按序号换行（每个机构一行）：  
     - `1. 机构/学校A`
     - `2. 机构/学校B`
     - `3. ...`

3. **发表年份**  
   - 格式：`YYYY.MM`  
   - 从论文的接收/发表/在线发布时间中选一个最合适的  
   - 若只有年份，月份用 `01` 作为默认（如 `2026.01`），无法确定则写 `N/A`

4. **期刊/会议**  
   - 内容：`期刊/会议全名 + 等级信息`  
   - 示例：  
     - `Scientific Reports（IF 4.6，SCI Q1，综合类）`  
     - `Expert Systems with Applications（IF 7.5，SCI Q1，CCF C）`  
   - 如果等级信息不在文中，按你已知信息补充；完全不清楚时可写：`期刊/会议等级待查`
   - 期刊/会议期刊/会议全名和等级信息之间必须是"真实换行"（用于表格自动换行，等同 Excel 的 `Alt+Enter`）

5. **摘要**  
   - 必须覆盖 3 个要素：
     - **问题背景**：这篇文章要解决什么问题，处于什么场景/行业/任务
     - **解决方法**：提出了什么方法/框架/模型（核心思想）
     - **达到目标**：希望/实际达到什么效果（如提升精度、降低成本、提高效率等）
   - 输出时每个要素结束必须换行（每个要素一行，不能用字面量 `\n`）；建议格式如下（小标题加粗，内容不加粗）：  
     - `**问题背景**：...`
     - `**解决方法**：...`
     - `**达到目标**：...`

6. **研究方法&内容**  
   必须包含两个部分（可以用小标题或分行）：
   - **研究内容和方法**：  
     - 研究对象和任务是什么  
     - 采用了什么总体框架/模型结构/关键技术（如联邦学习、神经网络结构、损失函数、采样策略等）
   - **达到目的**：  
     - 上述方法是为了实现什么具体目标（如"在保护隐私前提下做 backorder 预测""改善不平衡数据上的预测准确率"等）
   - 输出时两个部分之间必须换行（每部分一行；不能用字面量 `\n`）；建议格式如下（小标题加粗，内容不加粗）：  
     - `**研究内容和方法**：...`
     - `**达到目的**：...`

7. **实验**  
   实验字段拆成三块内容，统一写在一个字段中，但结构要清晰：

   - **实验工具**：  
     - 深度学习/机器学习框架（如 PyTorch、TensorFlow、Sklearn）  
     - 硬件/平台（如 GPU 型号、云平台）  
     - 其他关键工具（如 LIME/SHAP 等可解释性工具）
   - **数据集**：  
     - 所用数据集全名，如 `Predict Product Backorders (Kaggle)`  
     - 多个数据集用 `; ` 分隔
   - **评价指标**：  
     - 列出论文中主要使用的指标，如 Accuracy、Precision、Recall、F1-score、AUC-ROC、mAP 等  
     - 可在末尾补一句整体性能（如"最高准确率约 90.3%"）
   - 输出时每一块结束必须换行（每块一行；不能用字面量 `\n`）；建议格式如下（小标题加粗，内容不加粗）：  
     - `**实验工具**：...`
     - `**数据集**：...`
     - `**评价指标**：...`

8. **相关工作**  
   - 用**序号**形式按顺序列出 3–6 条主要相关工作：  
     - 每条包括：**作者/年份（可简写）+ 方法类型/技术 + 与本工作的差异或局限**  
   - 相关工作每条结束必须换行（每条一行；不能用字面量 `\n`）；建议格式如下：  
     - `1. 作者/年份 + 技术：...（差异/局限）`
     - `2. 作者/年份 + 技术：...（差异/局限）`
     - `3. ...`

9. **自我思考**  
   必须包含四部分，可以按分行或小标题写清楚：

   - **创新点**：这篇论文相对已有工作真正新的地方  
   - **优势**：在哪些方面表现更好（精度、效率、可解释性、工程可用性等）  
   - **局限**：数据规模、场景假设、实现难度、隐私/计算成本等方面的不足  
   - **启发**：对你后续研究/工程有什么启发或可借鉴的方向
   - 输出时每个部分结束必须换行（每部分一行；不能用字面量 `\n`）；建议格式如下（小标题加粗，内容不加粗）：  
     - `**创新点**：...`
     - `**优势**：...`
     - `**局限**：...`
     - `**启发**：...`

10. **有无代码**  
    - 若论文中给出代码链接或仓库地址：  
      - 直接写链接，如 `https://github.com/xxx/yyy`  
    - 若明确说明不开源或未给链接：  
      - 填写：`论文中无`  
    - 若提到"可向作者索取代码"等模糊信息：  
      - 写：`论文中未给出公开代码链接`

## 工作流程

### 1. 与用户确认输入

至少确认以下信息：

- PDF 文件路径（一个或多个）
- 总表文件路径（建议为 xlsx，比如 `F:\论文\SUMMARY.xlsx`）
- 如用户已有固定表头，本 skill **优先尝试匹配现有列名**：
  - `标题 / 作者 / 发表年份 / 期刊或会议 / 摘要 / 研究方法&内容 / 实验 / 相关工作 / 自我思考 / 有无代码`  
  - 若表中列名略有差异，通过"中文关键字+英文关键字"进行模糊匹配；缺失的列自动在最后补上

### 2. 阅读论文重点位置

从 PDF 中优先抽取：

- 首页：标题、作者、机构/学校、期刊/会议名称、DOI、发表/接收时间
- 信息栏：年份与月份（用来构造 `YYYY.MM`）
- 摘要：用来写"摘要"字段
- 方法章节：用来写"研究方法&内容"
- 实验章节：工具、数据集、指标、结果
- 相关工作章节：按点提炼出 3–6 条代表性工作
- 文末"Data availability / Code availability / Acknowledgements"等：用来判断是否给出代码链接

### 3. 信息抽取与格式化

- 标题：  
  - 保留原文英文标题  
  - 给出准确中文翻译，用括号包裹：`English Title（中文翻译）`
- 作者（机构/学校）：  
  - 从作者信息中只提取机构/学校名称  
  - 多个机构按 `1. 2. 3.` 形式逐行输出
- 发表年份：  
  - 解析"Received / Accepted / Published / Online"中的最合适时间  
  - 转换为 `YYYY.MM`；若没有月份则默认 `01`
- 期刊/会议：  
  - 写出完整刊名/会名  
  - 如果你知道其 IF、分区、CCF 等信息，则加在括号内；否则可以简要标注领域和大致档次或写 `等级待查`
- 摘要：  
  - 明确写出"问题背景 + 解决方法 + 达到目标"三部分  
  - 每个要素结束都要插入"真实换行"（等同 Excel `Alt+Enter`）；小标题可用 `**...**` 加粗
- 研究方法&内容：  
  - 分成"研究内容和方法""达到目的"两段或两小节
  - 两段之间必须换行（每段一行）；小标题可用 `**...**` 加粗
- 实验：  
  - `工具：...` / `数据集：...` / `评价指标：...` 三部分都要出现
  - 每块结束都要换行（每块一行）；小标题可用 `**...**` 加粗
- 相关工作：  
  - 用 `1. 2. 3.` 序号逐条列出；每条结束都要换行
- 自我思考：  
  - 保证包含"创新点、优势、局限、启发"四个小节  
  - 每个小节结束都要换行（每小节一行）；小标题可用 `**...**` 加粗
- 有无代码：  
  - 如有链接直接写链接；无则写 `论文中无`

### 4. 追加到总表第一张表

- 在用户指定的 xlsx/csv 文件的 **第一张表** 中：
  - 若首行表头为空，则自动填入上述字段中文名作为表头  
  - 若已有表头，则按字段含义对齐现有列；缺少的列补在末尾
- 将当前论文形成的一行数据 **append 到最后一行**
- 若一次处理多篇 PDF，逐篇生成并按顺序多行追加
- 调用以下脚本完成写入（使用 Bash 工具），**将 `<PLUGIN_DIR>` 替换为本 skill 所在插件的根目录**：
  ```
  python <PLUGIN_DIR>/skills/paper-to-master-table/scripts/append_summary_row.py --xlsx "<用户指定xlsx路径>" --data-json '<JSON字符串>'
  ```

### 5. 对用户的输出

对每次调用，输出两块信息：

1. **本次生成的总表行内容**（按字段顺序列出各字段值，方便用户核对）
2. **表操作说明**：
   - 指明写入的文件路径
   - 说明是第一张表（如 `Sheet: LLMAgent`）
   - 追加的行号（例如 `Row: 22`）
